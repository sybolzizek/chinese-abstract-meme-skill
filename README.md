# 中文抽象梗 Skill

> [进入交互动效页](https://sybolzizek.github.io/chinese-abstract-meme-skill/)

让 Agent 在具体中文互联网语境里自然地玩梗、反串和说抽象话。

它不是一份“万能网络黑话清单”，也不是把几个短词随机塞进回复的提示词。这个项目把可核对的词条、变体和网友原话整理成语料，让 Agent 在需要时自己取用；接不上就正常说话。

> 还在施工。语料、例子和技能本身都接受纠错与补充。

## 能做什么

- 理解并使用中文互联网里仍在流通的梗、简称、谐音和固定句式。
- 在游戏、体育、数码、热点、日常聊天等具体场景里自然反串或吐槽。
- 问到词源、含义、变体时，读取对应词条和来源材料，而不是凭记忆现编。
- 在没有合适梗可用时，保持正常表达，不硬整。

## 不做什么

- 不把“抽象”写成客服话术或状态分类。
- 不把语料库当新闻、比赛、人物和热点的事实来源。
- 不保证每个词条都适合当前语境；老梗、过度日常化的词和低浓度白话会持续清理。
- 不替用户自动发表攻击、骚扰或仇恨内容。

## 快速开始

克隆仓库后，把整个目录放进你的 Agent/Skill 目录，或在支持本地 Skill 的客户端中选择这个仓库里的 `SKILL.md`。

```bash
git clone https://github.com/sybolzizek/chinese-abstract-meme-skill.git
```

最小可用形态只需要 `SKILL.md`。如果你的 Agent 能连接本项目后续公开的语料服务，再配置对应地址即可；没有语料服务时，Skill 仍可独立工作，只是不会获得持续更新的词条材料。

## 使用方式

安装后像正常聊天一样提问即可。不要先要求它“进入抽象模式”，也不必指定某个固定人设。

```text
帮我串一下这场比赛，但先把刚发生的事查清楚。
```

```text
这句话什么意思？它现在还能这么用吗？
```

```text
把这段回复改得像正常网友说的，别像产品文案。
```

真正起作用的是事实、语境、当前舆论和词条之间的联想；不是某个单独的万能词。

## 语料长什么样

每条词条至少有词语、解释和例子；衍生与来源可为空。

```json
{
  "term": "孝",
  "explanation": "……",
  "examples": ["……"],
  "derivations": ["孝子", "尽孝"],
  "sources": [
    {
      "title": "页面标题",
      "url": "https://example.com",
      "excerpt": "原文摘录"
    }
  ]
}
```

例子和变体不是模型替你编的“示范句”。它们应当能回到实际出现过的网页、帖子、评论或图文材料。

## 贡献语料

欢迎提交新词、补充旧词和纠正来源，但请不要只丢一个词。

- 给出词条的常见写法。
- 说明它在什么语境里被使用；不确定的部分直接标不确定。
- 至少附一条实际原文用法。
- 变体只收真实出现过的写法。
- 能给链接、截图或可复核出处就一起给。

词条不是越多越好。已经完全融入日常语言、只剩历史价值、脱离场景就没有意味的内容，会被保留为资料或从可用语料中移出。

## 项目状态

目前在做三件事：

1. 清洗已有语料，补全真实例句、变体和来源。
2. 建立可持续更新的公开语料库与审核流程。
3. 让更多 Agent 通过公开接口读取已确认语料，而不是复制一份僵死提示词。
